# Out-of-fold stacking produces a meta-learner fit on leakage-free base predictions

## Statement

Let $M$ base classifiers be trained inside a K-fold cross-validation scheme. For each sample $i$ and base model $m$, let $\hat{p}_{i,m}^{\text{OOF}}$ be the prediction produced when sample $i$ was in the held-out fold. Stack the OOF predictions into a design matrix

$$Z \in \mathbb{R}^{n \times M}, \qquad Z_{i,m} = \hat{p}_{i,m}^{\text{OOF}}.$$

The stacking meta-learner is a logistic regression with L2 penalty fit on $(Z, y)$:

$$\hat{p}_i^{\text{stack}} = \sigma\!\left( \beta_0 + \sum_{m=1}^{M} \beta_m \, Z_{i,m} \right), \qquad \sigma(z) = \frac{1}{1 + e^{-z}}.$$

Weights $\beta$ are estimated by penalized maximum likelihood:

$$\hat{\beta} = \arg\min_{\beta} \; - \sum_{i=1}^{n} \left[ y_i \log \sigma(\beta^\top z_i) + (1 - y_i) \log\bigl(1 - \sigma(\beta^\top z_i)\bigr) \right] + \lambda \|\beta\|_2^2.$$

Optional isotonic calibration $h$ is applied post-stack: $\hat{p}_i^{\text{final}} = h(\hat{p}_i^{\text{stack}})$.

## Derivation

Wolpert (1992) showed that a higher-level learner trained on base-learner predictions can reduce generalization error relative to any single base learner, provided the base-learner inputs to the meta-learner are unbiased. "Unbiased" here means the base-learner prediction for row $i$ must come from a model that never saw $y_i$ during training. Out-of-fold predictions satisfy this by construction: in K-fold CV, each row is held out exactly once, so its prediction is generated by a model trained on the complementary $K-1$ folds.

Breiman (1996) and Van der Laan et al. (2007, Super Learner) formalized the argument: the OOF stack $Z$ has the same distribution at train and test time for the meta-learner, which is the condition for standard statistical learning guarantees to apply.

## Boundary conditions

- Applies only when fold assignments are identical across base models. Different random seeds per base model violate the row-alignment assumption and invalidate the stack.
- Applies only when base models receive identical preprocessing (e.g., same imputation, same scaler, same downsample ratio). Otherwise $Z_{i,m}$ encodes preprocessing variation rather than model diversity.
- Does NOT apply when the meta-learner is fit on in-fold predictions. The anti-pattern is fitting base models on the full training set and then using those in-sample predictions as $Z$; the meta-learner will assign near-unit weight to whichever base model overfits hardest. See [[condensates/stacking-requires-oof-not-infold]].
- Post-stack isotonic calibration is optional. When applied, it MUST be fit on held-out predictions relative to the stacking fit (a second CV loop), otherwise the calibrator inherits the meta-learner's optimism.

## Worked reference

$M = 4$ base models (LR_EN, LinSVM_cal, RF, XGBoost), $n = 148$ cases, 5-fold CV.

Each base model is trained on 4 folds and predicts the 5th. Repeating across all folds gives one $\hat{p}_{i,m}^{\text{OOF}}$ per (sample, model) pair. The design matrix $Z$ is $148 \times 4$. Meta-learner (L2 logistic regression, $\lambda = 1$) is fit on $(Z, y)$ and learns weights, e.g., $\beta = (\beta_0, 0.42, 0.31, 0.18, 0.09)$ — the meta-learner allocates most weight to the two linear base models, which had the best OOF discrimination on this cohort. Final probability for a new sample is the sigmoid of $\beta^\top z_{\text{new}}$ where $z_{\text{new}}$ is the 4-vector of base-model predictions on the new sample (now trained on the full training set).
